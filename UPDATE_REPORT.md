## 更新内容

### 1. 新增智能音箱控制功能 ✅

#### 支持的动作
| 动作 | 说明 | 状态 |
|------|------|------|
| `speaker_pause` | 暂停播放 | ✅ 可用 |
| `speaker_next` | 下一首 | ✅ 可用 |
| `speaker_previous` | 上一首 | ✅ 可用 |
| `voice_command` | 语音指令 | ✅ 可用（播报文字） |

#### 不支持的动件
| 动作 | 说明 | 原因 |
|------|------|------|
| `speaker_play` | 播放 | ❌ 音箱不支持（无播放队列） |

### 2. 文档更新 ✅

#### 新增内容
- 智能音箱控制章节
- 动作控制 vs 语音指令的区别说明
- 设备类型说明（音箱、灯、空调等）
- 故障排查章节（音箱控制问题）

#### 关键说明
- **语音指令** (`voice_command`): 音箱播报文字，不执行动作
- **动作控制** (`speaker_pause` 等): 真正控制播放状态
- **音箱音量**: 只读属性，无法通过 API 设置

### 3. 代码更新 ✅

#### 更新的文件
1. **SKILL.md** - 完善文档，添加音箱控制说明
2. **cli.py** - 添加音箱动作支持
3. **channels/xiaomi.py** - 实现音箱控制逻辑

## 使用示例

### 暂停音箱
```bash
home-device --yes control 2085693780 speaker_pause
```

### 下一首
```bash
home-device --yes control 2085693780 speaker_next
```

### 发送语音指令（音箱播报文字）
```bash
home-device control 2085693780 voice_command --value "你好"
```

## 已知限制

### 音箱控制限制
1. **无法播放指定歌曲** - 需要语音控制或 App
2. **无法直接设置音量** - 音量属性只读
3. **不支持播放动作** - 音箱无播放队列

### 解决方法
- 使用语音指令开始播放：`voice_command` + "播放孙燕姿的歌曲"
- 使用物理按键或米家App调节音量
