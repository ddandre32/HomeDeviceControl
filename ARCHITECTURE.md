# XiaomiDeviceControl Skill - 功能与技术架构分析

## 📋 功能概览

XiaomiDeviceControl 是一个通过自然语言控制小米智能家居设备的 OpenClaw Skill，支持从简单开关到复杂组合控制的多种场景。

### 核心功能矩阵

| 功能类别 | 具体功能 | 实现状态 |
|---------|---------|---------|
| **设备查询** | 列出所有设备、按条件筛选、查看设备详情 | ✅ |
| **基础控制** | 开关、亮度调节、温度设置、颜色设置 | ✅ |
| **批量控制** | 多设备顺序/并行控制、延迟控制 | ✅ |
| **场景管理** | 列出场景、执行场景 | ✅ |
| **自然语言** | 中文/英文指令解析、模糊匹配 | ✅ |
| **高级功能** | 摩斯密码发送、延迟任务 | ✅ |

### 2. 模块详解

#### 2.1 自然语言解析模块

**核心类**: `XiaomiDeviceControlSkill.natural_language_control()`

**解析流程**:
```
用户指令 → 预处理(小写/去空格) → 分类识别 → 参数提取 → 执行
```

**支持的指令模式**:

| 模式 | 正则表达式/关键词 | 示例 |
|------|------------------|------|
| 查询指令 | `有哪些\|列出\|查看\|list` | "查看有哪些设备" |
| 场景指令 | `执行\|场景\|模式` | "执行回家模式" |
| 延迟指令 | `(\d+)秒后?` | "5秒后关闭台灯" |
| 批量指令 | `和\|并\|同时\|一起` | "打开客厅灯和卧室灯" |
| 摩斯指令 | `摩斯密码\|morse` | "发送摩斯密码 SOS" |
| 基础指令 | `打开\|关闭\|亮度` | "打开客厅的灯" |

**设备名称提取算法**:
```python
# 1. 房间+设备模式
r'(客厅|卧室|书房)[的\s]*(.+?)(灯|开关|窗帘)'

# 2. 直接提取设备关键词
keywords = ["灯", "开关", "插座", "窗帘", "空调", "音箱", ...]

# 3. 清理动作词后返回
remove_words = ["打开", "关闭", "把", "将", "秒后", ...]
```

#### 2.2 设备管理模块

**核心方法**:
- `list_devices()` - 设备列表查询
- `get_device_info()` - 设备详情
- `control_device()` - 设备控制
- `_find_device_by_name()` - 模糊匹配

**设备类型推断**:
```python
type_keywords = {
    "light": ["light", "lamp", "灯"],
    "switch": ["switch", "开关", "插座"],
    "curtain": ["curtain", "窗帘", "电机"],
    "speaker": ["speaker", "音箱", "小爱"],
    # ... 共12种类型
}
```

**模糊匹配策略**:
1. 精确匹配：名称完全相等
2. 包含匹配：名称互相包含
3. 房间+类型：房间名+设备类型组合
4. 最短匹配：多个匹配时选名称最短的

#### 2.3 批量控制模块

**执行模式**:

| 模式 | 实现方式 | 适用场景 |
|------|---------|---------|
| 顺序执行 | for循环 + delay_after | "先开灯，5秒后关灯" |
| 并行执行 | threading.Thread | "同时打开所有灯" |

**代码结构**:
```python
def batch_control(operations, parallel=False):
    if parallel:
        # 多线程并行
        threads = []
        for op in operations:
            t = threading.Thread(target=run_op, args=(op,))
            threads.append(t)
            t.start()
        for t in threads:
            t.join()
    else:
        # 顺序执行
        for op in operations:
            result = control_device(...)
            if op.get("delay_after"):
                time.sleep(delay_after)
```

#### 2.4 摩斯密码模块

**摩斯码表**:
```python
MORSE_CODE = {
    'A': '.-', 'B': '-...', 'C': '-.-.', ...
    '0': '-----', '1': '.----', ...
    ' ': '/', '.': '.-.-.-', ...
}
```

**编码规则**:
- `.` (点) = 短信号：亮 0.2s，灭 0.2s
- `-` (划) = 长信号：亮 0.6s，灭 0.2s
- 字符间隔 = 0.6s
- 单词间隔 = 0.6s

**中文支持**: 通过 `CHINESE_TO_PINYIN` 映射表将中文转为拼音再编码

#### 2.5 XMIoT 通信模块

**通信流程**:
```
Skill → subprocess.run(["miot", "--format", "json", ...]) 
      → XMIoT CLI 
      → 小米云服务 API 
      → 智能设备
```

**命令映射**:

| Skill 操作 | XMIoT CLI 命令 |
|-----------|---------------|
| 列出设备 | `miot device list` |
| 获取设备信息 | `miot device get <did>` |
| 打开设备 | `miot device prop set <did> 2 1 true` |
| 关闭设备 | `miot device prop set <did> 2 1 false` |
| 设置亮度 | `miot device prop set <did> 2 2 <value>` |
| 执行场景 | `miot scene run <scene_id>` |

**错误处理**:
- `NOT_INSTALLED` - miot 未安装
- `NOT_AUTHENTICATED` - 未认证
- `TIMEOUT` - 命令超时
- `DEVICE_NOT_FOUND` - 设备不存在

## 🎯 关键技术点

### 1. 自然语言理解 (NLU)

**挑战**: 中文表达多样，同一意图多种说法

**解决方案**:
- 关键词匹配 + 正则表达式
- 多模式组合识别
- 模糊匹配容错

**示例**:
```
"打开客厅的灯" → turn_on + 客厅灯
"把客厅的灯打开" → turn_on + 客厅灯
"客厅灯打开" → turn_on + 客厅灯
```

### 2. 设备模糊匹配

**挑战**: 用户可能记不清设备完整名称

**解决方案**:
- 分层匹配策略（精确→包含→房间+类型）
- 最短名称优先（减少歧义）
- 类型推断辅助

**示例**:
```
用户说: "台灯"
匹配: "卧室台灯" (包含匹配)

用户说: "客厅的灯"
匹配: "客厅灯" (房间+类型)
```

### 3. 异步任务处理

**挑战**: 延迟控制和批量控制需要非阻塞执行

**解决方案**:
- `threading.Thread` 实现后台执行
- `time.sleep()` 实现延迟
- 守护线程避免阻塞主进程

### 4. 错误处理与恢复

**策略**:
- 分层错误码（NOT_INSTALLED → NOT_AUTHENTICATED → DEVICE_NOT_FOUND）
- 详细错误信息返回
- 建议性修复提示

## 📈 扩展性设计

### 可扩展点

1. **新设备类型**
   - 在 `_infer_device_type()` 添加关键词
   - 自动识别，无需修改其他代码

2. **新指令模式**
   - 在 `natural_language_control()` 添加识别逻辑
   - 复用现有执行方法

3. **新控制动作**
   - 在 `control_device()` 添加 action case
   - 映射到对应的 miot 命令

4. **定时任务**
   - 可集成 APScheduler
   - 或依赖系统 cron/launchd

## 📝 总结

XiaomiDeviceControl Skill 采用分层架构设计：

1. **表现层**: 自然语言解析，降低用户使用门槛
2. **业务层**: 设备管理、场景管理、任务调度
3. **通信层**: XMIoT CLI 封装，与小米云服务通信

**核心优势**:
- ✅ 自然语言交互，无需记忆命令
- ✅ 模糊匹配，容错性强
- ✅ 批量控制，支持复杂场景
- ✅ 摩斯密码等趣味功能
- ✅ 代码结构清晰，易于扩展

**技术亮点**:
- 正则表达式 + 关键词的多模式NLU
- 分层模糊匹配算法
- 线程池实现并行控制
- 完整的错误处理体系
